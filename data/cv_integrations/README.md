# Данные для CV-интеграций

Эта директория группирует датасеты по полям состояния, которые предсказывает
CV-модель и которые затем передаются в RL-pipeline.

Общие изображения лежат в `data/images/`, а общий файл меток — в
`data/labels.csv`. Каждый вариант ниже задаёт только свой `metadata.json`: путь
к общим данным и список `target_columns`, которые dataloader должен брать из
`labels.csv`.

`data/images/` и `data/labels.csv` являются локально генерируемыми файлами и не
коммитятся в git. Для восстановления датасета запустите
`notebooks/00_data/01_generate_images.ipynb`.

- `x_y/`: активный target только для позиции.
- `x_y_theta/`: активный target для позиции и угла.
- `x_y_theta_velocity/`: будущая заготовка для target'ов позы и скорости.
- `full_state/`: будущая заготовка для полного state-вектора в стиле LunarLander.
